Digital Materiality

Project Overview

This project explores digital materiality through the creation of generative visual artifacts. Instead of producing a single final image, the work investigates how images emerge through systems, variation, and transformation. These visuals are then used in a projection mapping installation.

Approach

A rule-based system was constructed to generate outputs by modifying parameters such as form, density, color, and motion. Each artifact is not individually designed, but emerges from a network of relationships within the system.

The process started with simple images, introducing variation through noise and material transformations. It then expanded into instancing and point cloud techniques to explore more complex spatial transformations. I'm just obsessed with point cloud effects...

Variations

Category 1

A 2D system was made using simple shapes and noise. Edge and displacement effects were added, and feedback loops made the image change over time. Ramp and HSV nodes were used to add color.

Reference: In-class exercise

Category 2

Based on the Category 1 node setup, a circle node was added, and HSV Adjust was used to modify the color, creating a new form.

Reference: In-class exercise

Category 3

Sound input was analyzed using CHOP nodes and converted into data. This data was used to control visual changes in the system. The visuals were built using noise, edge, displacement, and feedback nodes, creating patterns that respond to sound over time.

Reference Tutorial: YouTube Tutorial

Category 4

A point generator was used to create the initial structure. LFO CHOPs controlled time-based changes, while noise and feedback nodes generated evolving patterns. A bloom effect was added to enhance brightness and visual intensity.

Reference Tutorial: YouTube Tutorial

Category 5

A sphere was created and changed using noise to make an organic shape. It was then rendered with lighting and a camera.

A box geometry was used, and noise-driven point transformations created dynamic motion. The result was rendered in a 3D scene.

Reference: In-class exercise

Category 6

A point-based particle system was built and rendered in 3D, with audio input controlling the movement and visual effects enhancing the final result. Multiple video inputs were used, and variations in box scale and lighting were applied to create different outcomes.

Reference Tutorial: YouTube Tutorial

more poinclouds...

A particle system was built using video input and motion detection. The video was processed using threshold and optical flow to capture movement, which was then used to control particle behavior. Point transform and noise were applied to create dynamic motion, while feedback and bloom effects enhanced the visual output.

Projection Mapping

I experimented with projection mapping on multiple surfaces using Kantan Mapper. It was more difficult than expected to match the projection to the shape of the sofa, and working in the dark hurt my eyes!! I projected onto the floor to create the feeling that Mario is jumping on it. I also tested projecting onto a window, but it didn't work well because it's transparent, so I tried mapping onto a refrigerator instead 😆

Reflection

I had always wanted to try projection mapping, but my previous attempts didn’t work well, so I couldn’t fully develop it. In this class, I was able to properly learn and apply it, which was really valuable.

TouchDesigner has always been an interesting tool to me. I had been learning it gradually through YouTube tutorials, but this was my first time exploring it more seriously. What I find most appealing is that it allows complex systems to be built through simple node connections without writing code. Projection mapping expanded this further, as the image is no longer limited to a screen but interacts with physical space.
